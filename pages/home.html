Bem vindo(a) à plataforma de apoio aos testes automatizados
Objetivo e introdução ao uso da plataforma

  • Essa plataforma tem como objetivo disponibilizar os testes automatizados já implementados, permitindo executar as Features disponíveis através da aba "FEATURES" ou clicando aqui. Além de interligar a equipe que desenvolve internamente o robô de automação com os colaboradores que não possuem tal conhecimento, de forma a auxiliar no processo com a criação de casos de testes para o projeto de automação, enviando sua proposta de feature através da aba "SUGESTÕES" ou clicando aqui, utilizando a linguagem Gherkin no padrão BDD - Behavior Driven Development e também.

  • Como descrito acima, o projeto de automação Konviva segue os padrões da metodologia BDD que se baseia na escrita de cenários de testes chamados de features. Estes contém os requisitos e critérios de aceite do comportamento do sistema. Ela diz o que a funcionalidade precisa ter para ser iniciada, o que ela fará em seguida e quais serão os resultados após a sua execução. Leia mais sobre o assunto clicando aqui.

  • Para maiores informações de como executar um teste automatizado e/ou começar a contribuir com suas histórias de usuário, acesse a aba "INSTRUÇÕES" ou clicando aqui.

Internamente, nossos desenvolvedores utilizam as seguintes tecnologias para execução dos testes automatizados

utilizando a linguagem Gherkin para metodologia BDD

JavaScript
